# 虎嗅骨架五：事件深度追踪

适用场景：对一个正在发酵的热点事件进行多角度深度追踪，超越新闻报道表层。

---

## 结构模板

```
【开场：事件的核心冲突是什么】
用一句话点出这件事最本质的矛盾：
- 「[事件]的表面是[A]，但核心是[B]的博弈」
- 「这不是一个[大家说的问题]，而是一个[更深层的问题]」

【还原现场：事件的来龙去脉】
按时间线还原事件发展，但要有取舍：
- 只保留和「核心冲突」直接相关的节点
- 每个节点说明：谁做了什么决定，背后动机是什么

【多方立场梳理】
事件涉及的各方立场：
| 角色 | 立场 | 利益驱动 |
|------|------|---------|
| 方A | ... | ... |
| 方B | ... | ... |
| 监管方 | ... | ... |

【深层原因：为什么这件事会发生】
超越「事件本身」，分析结构性原因：
- 行业机制/市场结构为什么制造了这个冲突？
- 历史上有没有类似案例？结果如何？

【预判：接下来会发生什么】
基于现有信息做有限预判：
- 不同情景下的可能走向（A/B方案）
- 关键决策节点是什么？
- 谁掌握最终的决定权？
```

---

## 写作要点

- 时间线还原要完整但有取舍，不是罗列所有事件
- 多方立场梳理必须真实呈现各方合理性，不单边叙事
- 预判要有条件限定，不做绝对预测

## 适合题材举例

- 「[公司纠纷/收购/裁员]事件全记录：真相比你想的复杂」
- 「追踪[监管政策]：谁赢了，谁还在等」
- 「[产品/服务]争议背后的商业逻辑」
